Nabil, après avoir regardé ton code, je pense que le problème ne vient pas d'une petite erreur mais de la méthode utilisée. Ton outil fait ceci : PDF.js transforme chaque page en image Canvas. canvas.toDataURL() produit une très longue image Base64. html-docx-js tente d'intégrer ces images dans un DOCX. Or html-docx-js est ancien et gère mal les grosses images Base64, surtout avec les PDF multipages. Sur certains navigateurs, le DOCX est créé mais les images sont ignorées. Vérification rapide Dans le DOCX obtenu : Si tu vois "Page 1", "Page 2", etc. mais pas les images → c'est presque certainement html-docx-js. Si le document est complètement vide → il y a une erreur JavaScript avant la génération. Solution recommandée Au lieu de : utilise la bibliothèque docx.js : Elle permet d'insérer les images directement dans le document Word avec ImageRun, ce qui est beaucoup plus fiable.
